## ADDED Requirements

### Requirement: Parser produces processor output
Each parser SHALL be capable of producing a processed output that includes:
1. A Markdown file (`.md`) with the document content
2. A resource directory containing extracted images and attachments
3. A JSON mapping file (`resources.json`) linking placeholder IDs to original content

The processor output SHALL be generated by calling a `process()` method on the parser (or a dedicated processor class) after the `parse()` method produces the `IntermediateResult`.

#### Scenario: Parser produces all three output artifacts
- **WHEN** a parser's `process()` method is called with an `IntermediateResult` and an output directory
- **THEN** the output directory contains a `.md` file, a `resources/` directory with images, and a `resources.json` mapping file

### Requirement: Processor handles parser-specific output formats
Each parser's processor SHALL handle the unique characteristics of that parser's output:
- `python-docx`: Converts python-docx paragraph/table/image extraction into standardized Markdown
- `mineru`: Converts MinerU's `content_list_v2.json` element types into standardized Markdown with proper handling of formulas, code blocks, and images
- `pandoc`/`markdownit`: Converts pandoc-generated Markdown into the standardized format with XML placeholders

#### Scenario: MinerU processor handles formula elements
- **WHEN** the MinerU parser produces an element of type `formula`
- **THEN** the processor outputs a block-level `<formula id="N"/>` placeholder in Markdown and records the formula content in `resources.json`

### Requirement: Processor interface
The processor SHALL implement a standard interface:
```python
def process(self, intermediate: IntermediateResult, output_dir: Path, config: ProcessorConfig) -> ProcessResult
```
Where `ProcessResult` contains paths to the generated Markdown file, resource directory, and JSON mapping file.

#### Scenario: Processor is called with valid intermediate result
- **WHEN** `process()` is called with a valid `IntermediateResult` and writable output directory
- **THEN** processing completes without error and returns valid `ProcessResult` with file paths
